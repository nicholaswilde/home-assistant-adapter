# Specification: Dynamic Wifi AP SSID with Abbreviated MAC Address

## Overview
This feature updates the SSID of the captive portal Access Point (AP) generated by the ESP32-C3 when it cannot connect to the configured WiFi. Currently, the AP SSID is hardcoded as `HA-Adapter-<deviceId>`. This track will modify it to dynamically construct the AP SSID using the configured `deviceId` (acting as the hostname) from `Config.h` followed by a dash and the last 3 octets (6 hex characters) of the device's MAC address in uppercase, formatted as `<deviceId>-<short mac address>` (e.g. `oven-A1B2C3`).

## Functional Requirements
1. **Dynamic SSID Generation**:
   - Query the device's WiFi MAC address.
   - Extract the last 3 octets (6 hex characters) of the MAC address.
   - Convert the extracted hex characters to uppercase (e.g., `A1B2C3`).
   - Retrieve the `deviceId` from `Config.h`.
   - Concatenate `<deviceId>`, `-`, and the formatted `<short mac address>` to form the AP SSID.
2. **Access Point Initialization**:
   - Use the dynamically generated SSID string when starting `WiFiManager` autoConnect portal.
   - Example: if `deviceId` is `oven` and the last three bytes of the MAC address are `1A:2B:3C`, the AP SSID must be `oven-1A2B3C`.

## Non-Functional Requirements
- **Memory Safety**: Ensure no buffer overflows or memory leaks occur during dynamic string construction (e.g., utilizing safe C++ Arduino `String` manipulation).
- **Performance**: MAC address retrieval and string formatting should happen quickly during the configuration phase of setup.

## Acceptance Criteria
1. The firmware compiles successfully with strict compiler flags (`-Wall -Wextra -Werror`).
2. When the device starts `WiFiManager` captive portal due to WiFi connection failure, it starts an Access Point with the SSID formatted as `<deviceId>-<short mac address>` in uppercase (e.g., `oven-1A2B3C`).
3. The existing MQTT client connection and Home Assistant bridge initialization still use the original `deviceId` and operate correctly.

## Out of Scope
- Configuring or modifying the WiFi DHCP hostname of the ESP32.
- Modifying the MQTT client identifier formatting.
